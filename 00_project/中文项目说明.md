# 中文项目说明：每个文件夹和文件放什么

这个仓库是你的法国实习报告工作台。核心原则是：

- 你用中文思考、修改、补充细节；
- 我根据你的中文内容改写成正式法语；
- GitHub 保存轻量、可追踪、可恢复的写作文件；
- 大型原始资料不直接放进 Git，只在资料索引里记录路径。

## 总体结构

```text
rapport du stage/
  README.md
  .gitignore
  00_project/
  01_sources/
  02_notes/
  03_manuscript/
  04_figures/
  05_appendices/
  06_outputs/
  scripts/
```

## 根目录

### `README.md`

项目入口说明。放最简短的信息：项目主题、快速命令、主要目录。

不要在这里写正文，也不要放大量资料解释。

### `.gitignore`

告诉 Git 哪些文件不要上传，例如：

- Word/PDF 输出文件；
- 临时文件；
- 大型原始资料；
- Office 自动锁文件。

一般不需要你手动修改。

## `00_project/`：项目管理区

这里放“怎么组织这个项目”的文件，不放正文。

### `00_project/project_design.md`

项目设计总说明，解释整体结构、Git 策略、写作逻辑和中法双语原则。

适合放：

- 项目为什么这样组织；
- 哪些文件进 Git，哪些不进 Git；
- 中文稿和法语稿如何配合。

### `00_project/workflow.md`

日常工作流说明。

适合放：

- 每天怎么开始；
- 怎么写中文；
- 怎么生成法语 Word；
- 怎么同步 GitHub。

### `00_project/中文项目说明.md`

你现在正在看的文件。它用中文解释每个文件夹、每个文件应该放什么。

### `00_project/decision_log.md`

决策记录。

适合放：

- 为什么选择 Markdown 写作；
- 为什么大文件不进 Git；
- 为什么报告采用某个结构；
- 之后导师建议导致的结构变化。

格式建议：日期、决策、原因、影响。

### `00_project/task_board.md`

任务板。

适合放：

- 下一步要做什么；
- 正在做什么；
- 已完成什么。

它不是正式正文，只是项目进度管理。

## `01_sources/`：资料索引区

这里不存大型原始资料，主要记录“资料在哪里、用于哪一章”。

### `01_sources/source_inventory.md`

资料总索引。

适合放：

- Google Doc 链接；
- 本地 Word/PDF/Excel 路径；
- QGIS 项目路径；
- 哪个资料用于哪一章。

每次你新增一个重要资料，先登记到这里。

### `01_sources/bibliography_working.md`

临时参考文献列表。

适合放：

- 可能会用的文献；
- 法国机构资料；
- CEREMA、ADEME、INSEE、PLU 等官方资料；
- 还没完全统一格式的参考文献。

最终格式化后的参考文献放到 `03_manuscript/fr_final/07_bibliographie.fr.md`。

### `01_sources/data_dictionary.md`

数据字典。

适合放：

- 人口指标是什么意思；
- 热环境指标是什么意思；
- GVI、SVF、ICU、LCZ 等缩写解释；
- 每个指标来自哪里、能支持什么论点。

这里放中间笔记，不是最终正文。

## `03_manuscript/`：正文区

这是最重要的目录。以后正文都在这里管理。

它分成两套：

```text
03_manuscript/
  zh_working/   中文工作稿
  fr_final/     法语提交稿
  report_full.md
```

### `03_manuscript/zh_working/`

中文工作稿。你主要改这里。

适合放：

- 你的中文段落；
- 中文解释；
- 中文补充事实；
- 你不确定的想法；
- 对我说“这里帮我改成法语”的内容。

这些文件不是提交给学校的最终文本，但它们是法语稿的依据。

文件对应关系：

| 中文文件 | 内容 |
|---|---|
| `00_front_matter.zh.md` | 封面、致谢、目录等前置部分的中文说明 |
| `01_introduction.zh.md` | 中文版引言 |
| `02_partie_1_structure_territoire.zh.md` | 第一部分：机构和 Antibes 领土背景 |
| `03_partie_2_missions.zh.md` | 第二部分：CCTP/CCAP 和社会人口 SIG 任务 |
| `04_partie_3_analyse_innovation.zh.md` | 第三部分：数据局限、street view、IA、GVI |
| `05_partie_4_bilan.zh.md` | 第四部分：个人总结 |
| `06_conclusion.zh.md` | 中文版结论 |
| `07_bibliographie.zh.md` | 中文备注版参考文献 |

### `03_manuscript/fr_final/`

法语正式稿。这个目录用于生成 Word。

适合放：

- 正式法语正文；
- 学术语气；
- 最终标题；
- 最终参考文献；
- 可以直接交作业的内容。

你也可以直接修改这里，但推荐流程是：先改中文，再让我同步法语。

文件对应关系：

| 法语文件 | 内容 |
|---|---|
| `00_front_matter.fr.md` | Page de couverture, remerciements, sommaire |
| `01_introduction.fr.md` | Introduction générale |
| `02_partie_1_structure_territoire.fr.md` | Partie I |
| `03_partie_2_missions.fr.md` | Partie II |
| `04_partie_3_analyse_innovation.fr.md` | Partie III |
| `05_partie_4_bilan.fr.md` | Partie IV |
| `06_conclusion.fr.md` | Conclusion générale |
| `07_bibliographie.fr.md` | Bibliographie finale |

### `03_manuscript/report_full.md`

自动生成的完整合并稿。

不要手动改它，因为运行 `build_docx` 时会覆盖。

## `06_outputs/`：输出区

这里放生成的 Word/PDF。

适合放：

- `rapport_stage_antibes_draft.docx`
- 之后可能生成的 PDF；
- 临时交给导师看的版本。

这个文件夹不会上传 GitHub，因为输出文件可以重新生成。

## `scripts/`：脚本区

这里放可运行命令。

### `scripts/status.cmd`

查看项目状态。

运行：

```powershell
.\scripts\status.cmd
```

它会显示：

- Git 当前状态；
- GitHub 远程地址；
- 正文文件列表。

### `scripts/build_docx.cmd`

生成 Word 草稿。

运行：

```powershell
.\scripts\build_docx.cmd
```

它会读取 `03_manuscript/fr_final/` 里的法语稿，生成：

```text
06_outputs/rapport_stage_antibes_draft.docx
```

### `scripts/sync.cmd`

同步 GitHub。

运行：

```powershell
.\scripts\sync.cmd
```

它会：

1. 显示改动；
2. 让你输入 commit message；
3. 提交；
4. 从 GitHub 拉取最新版本；
5. 推送到 GitHub。

## 推荐工作方式

### 如果你要改正文

1. 打开 `03_manuscript/zh_working/` 对应章节；
2. 用中文写清楚你想表达的内容；
3. 告诉我“根据中文更新法语版”；
4. 我修改 `03_manuscript/fr_final/`；
5. 运行 `build_docx.cmd` 生成 Word。

### 如果你新增资料

1. 不要直接复制大文件进仓库；
2. 在 `01_sources/source_inventory.md` 增加路径；
3. 在 `02_notes/` 写一份摘要或提取；
4. 再决定是否写进正文。

### 如果你新增图

1. 原始图先留在原目录；
2. 最终选中的图复制到 `04_figures/selected/`；
3. 在 `04_figures/captions.md` 写图题和来源；
4. 正文里引用它。

## 最重要的原则

中文稿负责“想清楚”，法语稿负责“交作业”。

你不用一开始就写很完美的法语。你可以用中文把事实、感受、逻辑、例子写出来，我再帮你变成符合法国实习报告风格的法语文本。
